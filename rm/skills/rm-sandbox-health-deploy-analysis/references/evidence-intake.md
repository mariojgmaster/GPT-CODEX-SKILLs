# Evidence intake

Request and normalize evidence before diagnosing:
- environment table with role, name, URL, Id, and acting user
- validate/deploy output
- Flosum release logs, promotion errors, package references
- metadata/package diff or release note when available
- screenshots only when text export is unavailable

For multi-environment cases, require clear labels such as:
- `source`
- `target`
- `reference`
- `validation_target`

If the evidence set is incomplete, list the exact missing items before drawing conclusions.
