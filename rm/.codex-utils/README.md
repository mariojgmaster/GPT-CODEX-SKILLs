# .codex-utils policy

Use `.codex-utils/` for temporary helper artifacts generated by Codex inside the `rm` workspace.

Rules:
1. Never write helper files to the analyzed project or workspace root.
2. Use `.codex-utils/session-lock/session-lock.json` for read-only session lock state.
3. Keep ephemeral output under `.codex-utils/tmp/` or service-specific subfolders.
4. Use `.codex-utils/doc-output/` for explicitly requested DOCX/PDF deliverables.
5. Do not store RM case history in `.codex-utils`; use `.rm-logs/` for that purpose.
6. Do not version temporary helper files unless a future policy explicitly allows it.
