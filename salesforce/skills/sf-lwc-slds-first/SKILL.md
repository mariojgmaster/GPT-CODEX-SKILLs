# SF LWC SLDS First

Use this skill for Salesforce frontend tasks.

## Required flow
1. Confirm UI scope and existing component boundaries.
2. Apply SLDS-first from `references/lwc-rules.md`.
3. Keep client logic lightweight and secure.
4. Preserve component composition and naming patterns.
5. Route validation to `sf-validation-dryrun-targeted`.

## Hard rules
- Prefer SLDS utility and blueprint patterns.
- Keep accessibility and keyboard flow.
- Avoid unnecessary rerenders and heavy client processing.
- Avoid business-critical security logic only on client side.
